# restore\_urls.py

## 概要

`assign_filenames.py` の逆操作を行うユーティリティスクリプトです。

Excel内のI列（ファイル名）とF列（URL）の対応関係を構築し、G列・H列に記載された `[ファイル名|テキスト]` を `[URL|テキスト]` に復元します。同時にI列をクリアします。

これにより、Excelを `assign_filenames.py` 実行前の状態（ステップ2完了時点）に近い状態に戻し、AIエージェントに再投入して内容を修正・追記させることが可能になります。

## 背景と目的

ワークフローのステップ2でAIエージェントがExcelのE〜H列を記入した後、ステップ3で `assign_filenames.py` がURL→ファイル名の変換を行います。しかし、AI出力の内容に誤りがあった場合や、追加の要件行を記入した後に再度AIにE〜H列を修正させたい場合、G列・H列の `[ファイル名|テキスト]` 形式のままではAIが正しく処理できません。

本スクリプトは、ファイル名をURLに戻すことで、AIエージェントが再度参照・修正できる形式にExcelを復元します。

## ワークフロー上の位置づけ

本スクリプトはメインのワークフロー（ステップ1〜5）には含まれていません。ステップ3の操作を巻き戻す目的で任意に使用するユーティリティです。

```
（通常フロー）
ステップ2完了 → ステップ3 (assign_filenames.py) → ステップ4〜5

（やり直しフロー）
ステップ3以降完了後 → restore_urls.py で復元 → AIに再投入（ステップ2′） → ステップ3〜5 再実行
```

## Excel列構成

| 列 | 内容 | 本スクリプトでの扱い |
|---|---|---|
| A | 大分類 | 参照しない |
| B | 中分類 | 参照しない |
| C | 項番 | 参照しない |
| D | 要件内容 | 参照しない |
| E | 回答 | 参照しない |
| **F** | **証跡URL** | **読み取り（ファイル名→URL対応表の構築に使用）** |
| **G** | **証跡テキスト原文** | **読み書き（`[ファイル名\|テキスト]` → `[URL\|テキスト]` に置換）** |
| **H** | **日本語訳** | **読み書き（`[ファイル名\|テキスト]` → `[URL\|テキスト]` に置換）** |
| **I** | **ファイル名** | **読み取り→クリア（対応表構築後にセル値を消去）** |
| J | 索引 | 参照しない |

## 処理の詳細

### 1. ファイル名→URL対応表の構築（パス1）

2行目から最終行まで、F列とI列を同時にスキャンします。F列からURLを正規表現で抽出し、I列から改行区切りのファイル名を取得します。URLとファイル名を1対1で `zip` し、ファイル名→URLの対応辞書を構築します。

同一ファイル名が複数回出現した場合、最初に見つかったURLが使用されます（`assign_filenames.py` が同一URLに同一ファイル名を割り当てる仕組みなので、通常は全て同じURLです）。

構築結果はプレフィックス別にグループ化して表示されます。

### 2. G列・H列の置換とI列のクリア（パス2）

`[ファイル名|テキスト]` パターン（例: `[ZBX001|テキスト]`）をG列・H列内で正規表現で検出し、ファイル名部分を対応するURLに置き換えます。

```
置換前: [ZBX001|By default, Zabbix has four user roles...]
置換後: [https://www.zabbix.com/documentation/current/en/manual/config/users|By default, Zabbix has four user roles...]
```

パターン `[A-Z]+\d+` に一致するファイル名のみが置換対象です。対応表に存在しないファイル名はそのまま保持されます。

各行のI列は値をクリア（`None` に設定）します。

### 3. 安全な出力

元のExcelファイルは一切変更しません。処理結果は常に別ファイルに保存されます。

- `-o` 指定時: 指定したパスに保存
- `-o` 省略時: `{元ファイル名}_restored_{タイムスタンプ}.xlsx` として自動生成

## 前提条件と環境構築

### 必要な環境

Python 3.8以上がインストールされていること。

`assign_filenames.py` によるステップ3の処理が完了したExcelファイルが必要です。具体的には、F列にURL、I列にファイル名、G列・H列に `[ファイル名|テキスト]` 形式の値が記入済みであることが前提です。

### Python パッケージのインストール

```bash
pip install openpyxl
```

`openpyxl` はExcelファイル（.xlsx）の読み込みと書き込みに使用します。それ以外の依存パッケージはすべてPython標準ライブラリ（`argparse`、`re`、`sys`、`datetime`、`pathlib`、`collections`）のため、追加インストールは不要です。

## 使い方

### 事前確認（dry-run）

```bash
python restore_urls.py input_named_YYYYMMDD_HHMMSS.xlsx --dry-run
```

Excelを変更せず、ファイル名→URL対応表のみを表示します。初回実行時はまずこれで確認することを推奨します。

### 基本

```bash
python restore_urls.py input_named_YYYYMMDD_HHMMSS.xlsx
```

出力は `input_named_YYYYMMDD_HHMMSS_restored_YYYYMMDD_HHMMSS.xlsx` のような名前で同じディレクトリに保存されます。

### 出力先を明示

```bash
python restore_urls.py input_named_YYYYMMDD_HHMMSS.xlsx -o restored.xlsx
```

### 列位置のカスタマイズ

```bash
python restore_urls.py input.xlsx \
    --url-col G \
    --evidence-cols "H,I" \
    --name-col J
```

## コマンドラインオプション一覧

| オプション | デフォルト | 説明 |
|---|---|---|
| `excel`（必須） | – | 入力Excelファイル（assign\_filenames.py 処理済み） |
| `-o, --output` | 自動生成 | 出力Excelファイルのパス |
| `--url-col` | `F` | URL列 |
| `--evidence-cols` | `G,H` | 証跡テキスト列（カンマ区切り） |
| `--name-col` | `I` | ファイル名列 |
| `--dry-run` | – | 変更せず結果のみ表示 |

## 注意事項

1行目はヘッダー行として扱われ、処理対象外です。

G列・H列の置換は `[ファイル名|テキスト]` 形式（ファイル名が `[A-Z]+\d+` パターンに一致するもの）のみが対象です。それ以外の形式のブラケット表記は変更されません。

I列は全行の値がクリアされます（空白でない値を持つセルのみ）。クリアされた行数はサマリに表示されます。

本スクリプトはExcelの2行目からデータが存在する最終行までを自動的に処理します（`ws.max_row` を使用した動的検出）。
